# "창 닫으면 회의록 중단"의 근본 원인은 회의록 생성이 브라우저 트리거였던 것

- 잡(전사)은 이전에도 서버 백그라운드에서 완료됐지만, 회의록 마크다운 생성과
  cl_meetings 이력 저장은 잡 완료를 **폴링하던 브라우저가** /api/summarize 를 호출해야 실행됐다.
- 탭이 다시 열리지 않으면 전사는 DB에 있어도 회의록·이력은 영영 생성되지 않았다 — 증상("창 닫으면 중단")의 실체.
- 수정: lib/minutes.js 로 코어 분리 → lib/jobs-runtime.js 의 finalizeJob 이 서버에서
  교정→회의록→이력저장→원본 Drive 보관까지 완결. 클라이언트는 결과만 렌더(renderServerResult).
- 같은 부류의 회귀를 피하려면: "완료 후 후처리"를 클라이언트에 두지 말 것. 서버 잡의 단계로 넣고
  산출물 컬럼(corrected_text/minutes_md/meeting_id/audio_drive_id)을 체크포인트로 쓸 것.
