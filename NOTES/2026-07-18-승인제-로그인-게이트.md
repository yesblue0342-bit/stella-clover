# 승인제 로그인 인증 게이트 (컨텐츠 분리 없음, 접근만 차단)

- 요구: 누구나 접속·열람 가능하던 것을 막되(특히 CBO=개인 AI 키 연결), 1인 사용이라 컨텐츠 사용자별 분리는 하지 않음.
  → 서버 /api 게이트로 "승인된 로그인 세션"만 통과. 컨텐츠 API·DB 는 그대로(공유).
- 세션은 JWT 대신 DB 세션(cl_sessions) — 시크릿 불필요·취소 가능. 쿠키 clover_sid(HttpOnly, SameSite=Lax).
  Secure 는 프록시(NPM) 뒤 http 포워딩이라 생략(붙이면 컨테이너-프록시 구간에서 쿠키 유실 락아웃 위험). 브라우저-프록시는 https.
- 게이트는 server.mjs 미들웨어(app.use('/api'))로 CORS 다음·디스패치 이전에 배치. AUTH_PUBLIC=Set(['auth']) 만 공개.
  req.path 는 app.use('/api') 마운트 기준이라 '/auth' → sub='auth'(디스패치와 동일 계산) — 우회 없음.
- 관리자 시드: admin·yesblue0342 (둘 다 비번 admin, approved). ★배포 후 비번 변경 권장(auth password 액션).
- 서버 내부 워커/크론/노트동기화는 getPool 직접 호출(HTTP 미경유) → 게이트 영향 없음. cleanup 도 mock req 직접 호출이라 무관.
- 게이트가 DB 오류 시 401(안전 차단). 기존 앱은 로그인 후 정상 — 락아웃 방지 위해 관리자 시드가 항상 approved.
